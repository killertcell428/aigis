## はじめに

「Claude Codeでアプリ開発を自動化しました」——こういった記事は最近よく見かけるようになりました。

でも、サービスを世に出すために必要な作業って、コードを書くだけじゃないですよね。

- PyPIへのパッケージ公開
- Webサイトのデプロイ
- GitHubリポジトリの整備
- SNSアカウント作成・投稿
- 技術記事の執筆・投稿
- 運用の自動化スケジュール構築

今回、OSSプロジェクト「Aigis」のサービス展開を**開発からマーケティングまで、すべてClaude Code（Coworkモード）で自動化**してみました。本記事ではそのリアルな体験を共有します。

## Aigisとは

Aigisは、AIエージェントのセキュリティを強化するPythonライブラリです。プロンプトインジェクション検出、出力検証、レート制限などの機能を提供します。

> 本記事の主題はAigisの宣伝ではなく、**Claude Codeで「サービス展開の全工程」を自動化した実体験**です。

## 自動化した全工程の概要

今回自動化したのは、以下の全工程です。

| # | 工程 | ツール/サービス |
|---|------|----------------|
| 1 | Python開発・テスト | Claude Code |
| 2 | PyPIパッケージ公開 | twine + PyPI |
| 3 | Webサイト構築・デプロイ | Next.js + Vercel |
| 4 | GitHubリポジトリ整備 | GitHub |
| 5 | Googleアカウント作成 | Google |
| 6 | X（Twitter）投稿自動化 | Claude in Chrome |
| 7 | Qiita記事執筆・投稿 | Claude in Chrome |
| 8 | Zenn記事執筆・投稿 | Claude in Chrome |
| 9 | 定期実行スケジュール構築 | Cowork Scheduled Tasks |

通常なら数日〜数週間かかるこの一連の作業を、**1セッションで完了**させました。

## 各工程の詳細

### 1. Python開発・テスト

ここは多くの方が経験済みかと思います。Claude Codeにプロジェクトの要件を伝え、コードの生成・テスト実行・リファクタリングを繰り返しました。

特筆すべきは、テストカバレッジの向上もClaude Codeが自律的に行った点です。「テストを書いて」と言うだけで、エッジケースまで考慮したテストが生成されました。

### 2. PyPIパッケージ公開

`setup.py`の作成からビルド、`twine`によるPyPIへのアップロードまで、すべてClaude Codeのターミナルから実行しました。

```bash
# Claude Codeが実行したコマンド群
python setup.py sdist bdist_wheel
twine upload dist/*
```

バージョン管理やメタデータの設定も、対話的に確認しながら進めてくれるので、ミスが少ないです。

### 3. Webサイト構築・Vercelデプロイ

Next.jsでランディングページを構築し、Vercelにデプロイしました。

Claude Codeが担当したのは：
- レスポンシブなLPのコンポーネント設計
- ダークモード対応のスタイリング
- Vercel CLIを使ったデプロイ

ここで面白かったのは、デザインの微調整も「もう少しモダンな感じに」「CTAボタンを目立たせて」といった自然言語での指示で対応できた点です。

### 4. GitHubリポジトリ整備

README、CONTRIBUTING.md、Issue/PRテンプレートなど、OSSプロジェクトに必要なドキュメント類もClaude Codeが生成しました。

### 5. SNSアカウントの運用準備

ここからが本記事の本題です。**開発以外の業務の自動化**に入ります。

Claude Codeの「Claude in Chrome」機能を使い、ブラウザ操作を自動化しました。Xアカウントのプロフィール設定、自己紹介文の作成なども自動で行えます。

### 6. X（Twitter）投稿の自動化

Xへの投稿は、以下のような戦略を組み込んで自動化しました。

**投稿タイプのローテーション（7種）：**
- Tips/知見共有
- 開発進捗
- 質問/議論
- ニュース反応
- データ/事例共有
- コミュニティ連携
- ストーリー/体験談

Xのアルゴリズムを考慮し、リプライの重要性（通常ツイートの75倍の重み）や、投稿後30分間のエンゲージメント獲得が重要であることも加味しています。

### 7-8. 技術記事の執筆・投稿（Qiita/Zenn）

ここが最も技術的に面白い部分でした。

**課題：CodeMirror 6エディタへの入力**

QiitaもZennもCodeMirror 6（CM6）ベースのエディタを使用しています。通常のDOM操作（`element.value = "..."`）では入力できません。

**解決策：クリップボードAPIを使ったペースト**

```javascript
const cm = document.querySelector('.cm-content');
const dt = new DataTransfer();
dt.setData('text/plain', articleContent);
const pasteEvent = new ClipboardEvent('paste', {
  bubbles: true,
  cancelable: true,
  clipboardData: dt
});
cm.dispatchEvent(pasteEvent);
```

`DataTransfer`オブジェクトにテキストを設定し、`ClipboardEvent`をディスパッチすることで、CM6エディタに確実にコンテンツを挿入できます。これはCM6を使ったサービス全般で応用可能なテクニックです。

**Qiitaでのハマりポイント：**
- タグ入力時のドロップダウン選択が必要
- 新規アカウントでの連続投稿制限（`/drafts/new`が`/trend`にリダイレクトされる）

### 9. 定期実行スケジュールの構築

最後に、これらの作業を継続的に回すためのスケジュールを構築しました。

| タスク | 頻度 | 内容 |
|--------|------|------|
| X投稿 | 毎日9時/18時 | トレンドに合わせた投稿 |
| 記事下書き | 毎週月曜9時 | トレンド調査→記事作成→投稿 |
| エンゲージメント | 毎日12時 | リプライ・いいね |
| 週次分析 | 毎週金曜17時 | 指標確認・改善 |

記事作成のスケジュールでは、単なる自社プロダクト紹介ではなく、**トレンド調査→読者に価値ある記事作成→自然な形で自社ツール紹介**という構造のPDCAを組んでいます。

## うまくいった点・つまずいた点

### うまくいった点

1. **一貫性のあるブランディング**: 同じセッション内で全工程を行うため、トーンやメッセージが統一される
2. **速度**: 手動なら数日かかる作業が、数時間で完了
3. **ドキュメント化が自動**: 作業ログが残るので、後から振り返りやすい

### つまずいた点

1. **CM6エディタとの格闘**: 標準のDOM操作が効かず、クリップボードAPI経由という回り道が必要だった
2. **サービスごとの制限**: Qiitaの新規アカウント制限など、自動化では予測しにくい壁がある
3. **認証フロー**: OAuthやCAPTCHAなど、完全自動化が難しい部分は残る

## まとめ

Claude Code（Coworkモード）を使えば、**コード開発だけでなく、サービス展開に必要な業務全体を自動化**できます。

特に以下のような場面で威力を発揮します：
- 個人開発者がOSSを公開するとき
- スタートアップが素早くMVPを市場に出したいとき
- マーケティング施策を含めたサービスローンチ全体を効率化したいとき

「コードを書く」だけでなく「サービスを届ける」ところまで、AIアシスタントと一緒に自動化する時代が来ています。
