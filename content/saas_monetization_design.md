## 3. Stripe 実装計画

### 使うべき Stripe 機能

| 機能 | 用途 |
|------|------|
| **Stripe Checkout** | サブスクリプション開始画面。Stripe がホストするため PCI 準拠が不要 |
| **Customer Portal** | ユーザーが自分でプラン変更・支払い方法更新・解約できる画面 |
| **Webhooks** | 支払い状態の変化をリアルタイムで受信し DB を更新する |
| **Stripe Metered Billing** | 超過リクエスト分の従量課金（フェーズ 2 以降） |
| **Stripe Trial Periods** | 14日無料トライアル。クレジットカード登録を先に取る |

---

### Subscription プランの設定方法

1. **Stripe Dashboard → Products → Add Product** で各プランを作成する
   - Product 名: "Aigis Pro", "Aigis Business"
   - Pricing: Recurring / Monthly
   - Price: $49.00 / $299.00

2. 作成された **Price ID** を環境変数に設定する:
   ```
   STRIPE_PRICE_PRO_MONTHLY=price_xxxxxxxxxxxxxxxx
   STRIPE_PRICE_BUSINESS_MONTHLY=price_yyyyyyyyyyyyyyyy
   ```

3. **Checkout セッション作成時**に Price ID を指定する（`stripe_client.py` 参照）

4. **Customer Portal** はダッシュボード上で有効化し、許可する操作（プラン変更・解約）を設定する

---

### 必要な Webhook イベント一覧

| イベント | トリガー | 処理内容 |
|---------|---------|---------|
| `checkout.session.completed` | 初回課金成功 | DB にプランを保存、ウェルカムメール送信 |
| `customer.subscription.updated` | プラン変更 | DB のプランを更新 |
| `customer.subscription.deleted` | 解約・期限切れ | プランを Free にダウングレード |
| `invoice.payment_succeeded` | 月次更新成功 | 継続確認ログ記録 |
| `invoice.payment_failed` | 支払い失敗 | 警告メール送信、猶予期間後にダウングレード |
| `customer.subscription.trial_will_end` | トライアル終了 3 日前 | 課金開始リマインドメール送信 |

> **重要**: Webhook エンドポイントは必ず `stripe.Webhook.construct_event()` で署名検証を行う。
> 署名シークレットは `STRIPE_WEBHOOK_SECRET` 環境変数で管理する。

## 5. チャーン防止施策

### オンボーディングチェックリスト

初回ログイン後にダッシュボード上で表示するウィザード形式のチェックリスト。完了率が高いほどチャーンリスクが低い。

```
[ ] APIキーを発行する
[ ] pip install aigis を実行する
[ ] 最初のスキャンを実行する（サンプルコードをコピー）
[ ] Slack通知を設定する
[ ] チームメンバーを招待する（Pro プランのみ）
```

完了率 80% 以上のユーザーは 30 日後の継続率が顕著に高い（一般的な SaaS 知見より）。

---

### 週次サマリーメール

毎週月曜 9:00 に前週のサマリーをメール配信する。

**メール構成:**
- 先週のブロック件数（前週比 +/-）
- リスクレベル別の内訳（CRITICAL / HIGH / MEDIUM）
- Top 3 の脅威カテゴリ
- 「先週ブロックしたことで守られた可能性があるもの」の説明文
- ダッシュボードへの CTA ボタン

**狙い**: プロダクトの価値を数字で可視化し、「使っていてよかった」という感覚を週次で強化する。

---

### In-app ヒント

ダッシュボード上で表示するコンテキスト依存のヒント。

| トリガー | ヒント内容 |
|---------|-----------|
| 初回ログイン後 | 「まずは APIキーを発行してみましょう → [発行する]」 |
| ブロック件数 0 のまま 3 日経過 | 「スキャンがまだ届いていません。SDK は導入できましたか？ → [クイックスタート]」 |
| CRITICAL イベント発生時 | 「深刻な脅威がブロックされました。詳細を確認してみましょう → [ログを見る]」 |
| Pro トライアル Day 10 | 「トライアル残り 4 日。継続することで ${今月のブロック数} 件の脅威から守り続けられます → [プランを続ける]」 |
| Business 機能（Human-in-the-Loop）を閲覧時 | 「この機能は Business プランで利用できます → [アップグレード]」 |
