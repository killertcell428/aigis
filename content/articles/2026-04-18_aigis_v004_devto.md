# I audited my own AI security product — here's what we found

Just shipped [Aigis](https://github.com/killertcell428/aigis) v0.0.4, an open-source firewall for AI agents. Two things landed in parallel:

1. **Internal security audit of the product itself** — Critical 4 / High 5 / Medium 4, plus 7 CodeQL alerts. All fixed.
2. **Seven 2025–2026 LLM-security research papers, one module each, zero deps.**

This post walks through the five most interesting findings from the audit, then shows how each of the seven papers maps into Aigis's existing defence layers.

---

## 5 audit findings worth showing

### 1. API key storage: unsalted SHA-256 + `==` (Critical)

```python
# before
def hash_api_key(raw_key: str) -> str:
    return hashlib.sha256(raw_key.encode()).hexdigest()

def verify_api_key(raw: str, stored: str) -> bool:
    return hash_api_key(raw) == stored
```

Two problems stacked: unsalted SHA-256 opens the door to rainbow-table recovery if the DB ever leaks, and raw `==` comparison leaks the stored hash one byte at a time via timing.

```python
# after
def hash_api_key(raw_key: str) -> str:
    return hmac.new(_hmac_key(), raw_key.encode(), hashlib.sha256).hexdigest()

def verify_api_key(raw: str, stored: str) -> bool:
    return hmac.compare_digest(hash_api_key(raw), stored)
```

HMAC keyed on the app secret, constant-time compare.

### 2. CORS: `*` origins + credentials (Critical)

`allow_origins=["*"]` + `allow_credentials=True` is a spec violation that quietly enables cross-origin authenticated requests through a victim's browser. Replaced with a settings-driven allowlist.

### 3. No Stripe webhook idempotency (Critical)

Stripe retries webhook delivery on failure. Without dedup, the same `checkout.session.completed` can apply the plan transition twice. Fix: add a `WebhookEvent(event_id UNIQUE)` ledger and `INSERT` right after signature verification — `IntegrityError` means we've already processed it.

### 4. Slack webhook had no SSRF guard (High)

`tenant.slack_webhook_url` is admin-controllable and was posted to without validation. `http://169.254.169.254/...` was a free gift. Fix: allowlist `hooks.slack.com` over HTTPS and reject private/loopback/link-local/reserved/multicast resolutions.

### 5. Custom-rules regex had no ReDoS guard, silently swallowed errors (High)

A tenant could upload `(a+)+b` and freeze the scanner, or upload malformed patterns in bulk to silently disable detection (`except re.error: continue`). Fix: new `_regex_guard.safe_compile_user_regex()` that rejects oversized patterns, nested quantifiers, and quantified alternation groups. Invalid rules now surface as `invalid_rule` matches so operators can see what's broken.

The other eight findings (JWT algo/claims pinning, super-admin gate on tenant creation, production validators for `SECRET_KEY` and DB password, bcrypt rounds=14, expanded confusables, layered docs exposure, etc.) plus the 7 CodeQL alerts are all in the [v0.0.4 CHANGELOG](https://github.com/killertcell428/aigis/releases/tag/v0.0.4).

After all fixes: **940/940** aigis core + **61/61** backend tests green.

## Seven research papers, mapped into the existing layers

Aigis already had a 4-wall + L4–L6 defence layout. Rather than adding a parallel "recent research" section to the README, every new module slots into the existing frame.

### Wall 1 — Pattern Matching

**1. `judge_manipulation` category (15 patterns)**

Inspired by AdvJudge-Zero (Palo Alto Unit 42, 2026), which hit ~99% bypass against LLM-as-Judge evaluators. Patterns cover forced verdicts, rubric override, reward-hack phrasing, role swap, and meta-instructions addressed to the judge, in English and Japanese.

**2. MCP 3-stage coverage**

MSB ([arxiv:2510.15994](https://arxiv.org/abs/2510.15994), Oct 2025) classifies MCP attacks into discovery, invocation, and response. Aigis already had discovery (`scan_mcp_server`); v0.0.4 adds `scan_invocation()` and `scan_response()` — both recursively flatten JSON args/responses and run the shared scanner over each string field.

### Wall 2 — Semantic Similarity

**3. `fast_screen` (Mirror Design Pattern, Mar 2026)**

Character-trigram log-likelihood ratio scorer. Two corpora → per-trigram LLR → logistic squash into [0, 1]. Pure stdlib, sub-millisecond per call. Good as a first-line pre-filter before the heavier regex + corpus similarity pass.

**4. `memory.imitation_detector` (MemoryGraft, Dec 2025)**

Planted memory entries that *structurally imitate* the system voice slip past content-pattern scanners. Aigis now runs char-4-gram Jaccard between untrusted memory writes and operator-registered voice references; high similarity on untrusted sources flags an `ImitationFinding`.

### Wall 3 — Encoded Payload

Confusables expanded: Armenian, Hebrew, Arabic-Indic digits, Fullwidth Latin A–Z/a–z/0–9, zero-width and bidi control codepoints. Emoji stripping rewritten from a broad character class into a codepoint-range function (also fixes the CodeQL warning).

### Pre-Wall (new tier) — Input Shaping

**5. `structured_query` (StruQ + LLMail-Inject)**

`StructuredMessage` splits a prompt into `system` / `instruction` / `data` slots. The untrusted `data` slot is scanned for role tokens (`<|im_start|>`, `[INST]`, `### System:`, ChatML markers) and override phrases. Violations raise `BoundaryViolation` — the prompt never reaches the model.

**6. `rag_context_filter` (DataFilter + RAGDefender)**

Per-chunk policy. `strip` removes offending sentences, `block` drops the whole chunk. Fast-screen + role-token check + directive phrase check combined.

### L7 (new) — Goal-Conditioned FSM

**7. `spec_lang.fsm` (MI9, Aug 2025)**

```python
fsm = AgentStateMachine(
    goal="write_report",
    initial_state="planning",
    states={
        "planning": StateRule(name="planning",
            allowed_tools=frozenset({"search", "note"}),
            allowed_transitions=frozenset({"gathering"})),
        "gathering": StateRule(name="gathering",
            allowed_tools=frozenset({"search", "fetch_url"}),
            allowed_transitions=frozenset({"writing"})),
    },
    terminal_states=frozenset({"done"}),
)
monitor = FSMMonitor(fsm)
violation = monitor.observe_tool_call("shell:exec")
# FSMViolation(kind="forbidden_tool", ...)
```

`monitor/drift.py` already catches statistical drift ("this looks unusual"). The FSM catches declarative violations ("this was explicitly forbidden"). They compose.

## Design principle: no floating "what's new"

Every new module had to map onto the existing architecture or be rejected. A "Recent Research" section that lives outside the frame is a smell — it usually means the module doesn't actually fit, just got bolted on. Easier for users to reason about a product that has one picture and gains layers, than one that gains sections.
