## Comment template B: dev experience question for OSS dev tools

```
Great that you kept dependencies minimal — `npm install` / `pip install` for devtools that pull 200 transitive packages is something I actively avoid. Quick question: how do you handle upgrade compatibility when [core library] bumps majors? Do you pin or range?

Happy to see more zero-dep-first tools in 2026.
```
