## EN thread (shorter, 5 tweets)

```
1/ Shipped Aigis v0.0.4 — my OSS AI firewall just passed its own security audit.

13 internal findings (Critical 4 / High 5 / Medium 4) + 7 CodeQL alerts. All fixed, all documented in the CHANGELOG.

https://github.com/killertcell428/aigis/releases/tag/v0.0.4
```

```
2/ The worst one: API key hashing was unsalted SHA-256 with `==` comparison.

Rainbow-table-recoverable + timing-leakable. In a security product.

Replaced with HMAC(app_secret) + hmac.compare_digest. Not quietly — publicly, in the release notes.
```

```
3/ Also landed in v0.0.4: seven LLM-security research papers, one module each, zero dependencies.

Mirror, StruQ, MemoryGraft, MI9, MSB, DataFilter/RAGDefender, AdvJudge-Zero. Each module's docstring cites the paper.
```

```
4/ Key design choice: every new module had to map onto the existing 4-wall + L4-L6 defence layout.

No floating "Recent Research" section. If a feature can't be pinned to a layer, it doesn't fit the product, it just got bolted on.

L7 added. Pre-Wall tier added. Wall 1/2/3 gained new surfaces.
```

```
5/ pip install pyaigis==0.0.4 — zero deps, still.

An OSS firewall for AI agents, with a cleaner release model than anything else in the category (IMO, obviously).

https://github.com/killertcell428/aigis
```
